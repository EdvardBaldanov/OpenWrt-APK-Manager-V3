# История разработки OpenWrt APK v3 Manager (Next Gen)

## [18.03.2026] — Глобальное обновление: UI, GitHub и Оптимизация

### 1. Рефакторинг интерфейса (Glassmorphism Tabs)
- **Вкладочная структура**: Основной дашборд разбит на 4 функциональные вкладки:
  - **🏠 Обзор**: Статистика, кнопка принудительного обновления и **живой вывод системных логов**.
  - **📦 Пакеты**: Браузер файлов APK с поиском и информацией о размере/дате.
  - **🛰️ Источники**: Управление списком отслеживаемых GitHub-репозиториев.
  - **⚙️ Настройки**: Конфигурация токенов, интервалов синхронизации и архитектур.
- **UX Подключение к роутеру**: Блок инструкций вынесен в шапку страницы. Реализована **автоматическая подстановка IP-адреса сервера** прямо в команды для копирования.

### 2. Оптимизация для GitHub и CI/CD
- **Чистка бинарников**: Проведен хирургический аудит папки `bin/`. 
  - Удалено более 135 МБ лишних статических библиотек и заголовков.
  - Сохранены только критические runtime-компоненты для работы `apk-tools 3.0.2` на x86_64.
  - Итоговый размер папки сокращен со **144 МБ** до **8.6 МБ**, что позволяет использовать репозиторий в GitHub Actions.
- **Публикация**: Репозиторий успешно инициализирован и опубликован по адресу: [EdvardBaldanov/OpenWrt-APK-Manager-V3](https://github.com/EdvardBaldanov/OpenWrt-APK-Manager-V3)

### 3. Архитектура и надежность
- **Новая система конфигурации**: Настройки разделены на независимые JSON-файлы для безопасности и удобства бэкапа (`auth.json`, `web_auth.json`, `repos.json`, `filters.json`).
- **Миграция**: Реализован автоматический перенос настроек из старого `data/config.json`.
- **Отказоустойчивость**:
  - `check_auth` теперь поддерживает как новый, так и старый форматы учетных данных.
  - Реализовано неразрушающее сохранение настроек (merge вместо overwrite).
- **Авто-инициализация**: При первом запуске система сама создает необходимые директории и конфиги с дефолтными значениями.

### 4. Исправления (Hotfixes)
- Исправлено отображение кириллицы в логах и именах файлов.
- Восстановлена работа сервиса `Sync`, случайно затронутого при рефакторинге.
- Исправлена проблема "пустых конфигов" при перезапуске сервера.

## 🚀 Планы на будущее
- **Статическая сборка apk-tools**: Исследовать возможность сборки `apk-tools` в единый статически линкованный бинарный файл (Full Static Build). Это позволит полностью избавиться от папки `lib/` и сделать менеджер еще более компактным и переносимым.
- **Улучшение мониторинга**: Добавить графики активности и более детальную историю синхронизаций.
- **Поддержка GPG**: Возможность подписи не только через APK v3 (P-256), но и классическими GPG ключами для обратной совместимости.
